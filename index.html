    What Is an API?

    API = Application Programming Interface

    Restaurant Analogy

    1. You tell the waiter what you want
    2. Waiter sends request to kitchen
    3. Kitchen prepares food
    4. Waiter returns with the result

    Software Mapping

    • Customer = Mobile App
    • Waiter = API
    • Kitchen = Server
    • Food = Data

    Why APIs Exist

    Without APIs, companies would expose databases directly. That would be dangerous and unscalable.

    SecurityNo direct database access
    ConsistencySame format for every app
    ControlServer chooses what to share
    ScalabilityMillions of requests supported

    Client-Server Architecture

    Student Android App
    ->
    Battle Bots API
    ->
    Robot Database

    Client

    The app users run (Android app, browser, game).

    Server

    Stores data and processes requests (Google, Amazon, ECPI servers).

    What Happens During an API Request?

    1. User presses button
    2. Android app sends request
    3. API receives request
    4. Database is queried
    5. Data is returned
    6. App displays response

    This often happens in less than one second.

    HTTP Basics

    HTTP (HyperText Transfer Protocol) is the language for web and API communication.

    Browser/App  -- HTTP Request -->  Server
Browser/App <-- HTTP Response --  Server
    GETRetrieve information
    POSTCreate information
    PUTUpdate information
    DELETERemove information

    API Endpoints

    An endpoint is a specific API address or doorway into the server.

    https://battlebots.com/api/robots
    /api/robots/api/teams/api/matches/api/rankings

    JSON: The Common API Data Format

    JSON (JavaScript Object Notation) is lightweight, readable, and easy to parse.

    {
  "name": "Tombstone",
  "team": "Hardcore Robotics",
  "wins": 25
}
    [
  { "name": "Tombstone" },
  { "name": "Minotaur" },
  { "name": "Witch Doctor" }
]

    What Does "Consume an API" Mean?

    Using data provided by an API inside your application.

    1. Send request
    2. Receive JSON response
    3. Convert JSON into Java objects
    4. Display data in the UI

    Key Takeaways

    • APIs let applications communicate
    • Most API interactions follow the client-server model
    • HTTP carries requests and responses
    • JSON is a common data format
    • Consuming an API means retrieving and using server data
    • Battle Bots workflow: request -> JSON -> Java objects -> UI
